# 实时股票数据源调研

## 结论先行

当前决策：主数据源直接使用 **富途 OpenAPI**。其他数据源只作为后续备用方案评估。

2026-05-09 本机实测：

- OpenD 可连接，行情登录状态正常。
- 深市股票基础信息可获取，过滤出创业板股票 `1408` 只。
- `K_1M` 小批量订阅成功。
- 当前订阅额度为 `1000`，不足以一次性订阅全部创业板。

如果目标是“盘中实时监控创业板每只股票的 1 分钟 K 线并告警”，数据源优先级如下：

1. **富途 OpenAPI**：当前主方案。适合实时订阅、实时 K 线推送、快照和分时数据，工程形态最接近实时监控程序。
2. **Tushare Pro 实时分钟权限**：适合付费、稳定地拉取 A 股 1 分钟数据，接口清晰，适合作为主数据源或备用数据源。
3. **AKShare / 东方财富公开行情**：适合快速原型和低成本验证，但不建议作为最终稳定生产依赖。
4. **券商/商业行情 API**：如果需要更高可靠性、明确授权和更少限流风险，应作为长期方案评估。

## 1. 富途 OpenAPI

### 能力

富途 OpenAPI 支持：

- A 股行情数据。
- 实时订阅。
- 实时报价。
- 实时 K 线。
- 市场快照。
- 分时数据。
- 逐笔、摆盘等更细粒度数据。

对本项目最关键的是：

- `subscribe` 订阅股票和数据类型。
- `get_cur_kline` 获取已订阅股票的实时 K 线。
- `CurKlineHandlerBase` 接收实时 K 线推送。
- `get_stock_quote` 获取已订阅股票的实时报价。
- `get_market_snapshot` 获取市场快照。

### 优点

- 支持推送模式，不必完全依赖高频轮询。
- API 结构比较完整。
- 本机已有富途 OpenD 目录，接入成本可能较低。
- 适合后续扩展到逐笔、盘口、分时、板块等数据。

### 风险

- 需要安装并运行 OpenD。
- 需要账号和 OpenAPI 权限。
- A 股实时行情权限、订阅数量、频率限制需要实际测试。
- 如果要一次性订阅全部创业板股票，可能遇到订阅额度或性能限制。

### 建议验证项

- 能否连接本机 OpenD。
- 是否能订阅 `SZ.300xxx` / `SZ.301xxx` 股票的 1 分钟 K 线。
- 单连接可订阅多少只股票。
- 全创业板股票订阅是否会被限制。
- 推送延迟是否满足告警需求。

## 2. Tushare Pro 实时分钟

### 能力

Tushare 提供 A 股实时分钟接口：

- `rt_min`：获取 A 股实时分钟数据，支持 `1MIN`、`5MIN`、`15MIN`、`30MIN`、`60MIN`。
- `rt_min_daily`：获取单只股票当日开盘以来的历史分钟数据。
- `rt_k`：获取 A 股实时日线行情，支持通配符，例如创业板 `3*.SZ`。

### 优点

- 分钟线字段直接可用，包含 open、high、low、close、vol、amount。
- 对“1 分钟 K 线监控”这个需求很直接。
- 支持多个股票代码同时提取。
- 接口文档清楚。

### 风险

- 实时分钟数据是单独权限，通常需要付费开通。
- 全创业板股票每分钟拉取需要设计批量请求和限流。
- 如果追求秒级发现异动，单纯分钟接口可能不如推送型接口。

### 建议定位

- 如果富途订阅全创业板受限，Tushare 可以作为更稳的主方案。
- 如果只需要“每分钟结束后发现异动”，Tushare 非常合适。
- 如果需要“分钟内正在剧烈波动就提醒”，仍建议配合快照或推送数据。

## 3. AKShare / 东方财富公开行情

### 能力

AKShare 封装了东方财富等公开数据源：

- `stock_zh_a_spot_em`：单次返回沪深京 A 股实时行情。
- `stock_cy_a_spot_em`：单次返回所有创业板实时行情。

字段包括：

- 最新价
- 涨跌幅
- 成交量
- 成交额
- 振幅
- 最高
- 最低
- 今开
- 昨收
- 量比
- 换手率
- 涨速
- 5 分钟涨跌

### 优点

- 免费、上手快。
- 很适合第一版验证股票池、告警规则、通知链路。
- `stock_cy_a_spot_em` 可以直接拿创业板实时行情截面。

### 风险

- 不是交易所授权级别的正式行情服务。
- 接口可能变化、限流、失败或变慢。
- “实时”通常是网页行情意义上的实时，不一定满足严肃低延迟要求。
- 不是天然 1 分钟 K 线接口，需要用快照自行聚合分钟 K 线，或另找分钟数据接口。

### 建议定位

作为开发阶段的低成本原型数据源，不建议作为最终唯一数据源。

## 4. 商业行情 API / 券商 API

如果后续要长期稳定运行，尤其是高频监控、商业用途、多人使用或公开服务，需要评估正式授权行情源。

候选方向：

- 券商量化 API。
- Wind。
- 同花顺 iFinD。
- Choice。
- 掘金量化。
- 聚宽 JQData。
- 米筐。

这些服务通常比公开接口更稳定，但需要确认：

- 是否提供 A 股实时分钟或 tick 数据。
- 是否允许本地程序实时调用。
- 是否覆盖创业板全量股票。
- 请求频率和并发限制。
- 行情授权和费用。

## 5. 推荐实施路径

### 第一步：用 AKShare 快速跑通闭环

先用 `stock_cy_a_spot_em` 每 5-10 秒拉取一次创业板实时行情截面：

- 过滤创业板股票。
- 自行聚合 1 分钟 K 线。
- 实现异常波动检测。
- 打通控制台/日志/企业微信通知。

这样最快能验证产品逻辑。

### 第二步：验证富途 OpenAPI

测试富途是否可以稳定订阅创业板实时 K 线：

- 如果能全量订阅，富途作为主数据源。
- 如果只能订阅部分，富途可用于重点股票池，Tushare 或 AKShare 做全量扫描。

### 第三步：接入 Tushare 实时分钟作为稳定方案

如果接受付费，Tushare 的 `rt_min` 可以作为分钟 K 线主来源：

- 每分钟分批拉取创业板股票。
- 对返回分钟数据做异动检测。
- 用 `rt_min_daily` 做盘中补数据和恢复。

### 第四步：做数据源抽象

代码上不要绑定某一个数据源，建议抽象为：

```text
MarketDataProvider
  get_stock_universe()
  get_realtime_snapshot(codes)
  get_minute_bars(codes, freq)
  subscribe_minute_bars(codes, callback)
```

这样可以先用 AKShare，后续替换成富途或 Tushare。

## 6. 当前建议

本项目建议采用“双阶段数据源策略”：

- **原型阶段**：AKShare/东方财富创业板实时行情截面，低成本快速实现。
- **可用阶段**：优先富途 OpenAPI，如果全量订阅受限，则改为 Tushare 实时分钟权限。

最终生产运行不建议只依赖免费网页接口。
